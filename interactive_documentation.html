Windfield Real Estate & Development — Standardized Property Data Schema v1.0

The Property Profile as Data Foundation

The Master Property Profile serves as the standardized data container for every location in the Windfield analytical ecosystem, enabling consistent data treatment across all property types.

Purpose

Whether a property is a prospective site, competitor location, client existing location, or benchmark comparable—it receives the same structured data treatment. This enables:

  • Apples-to-apples comparison across disparate property types
  • Consistent data ingestion from multiple sources (CoStar, CenterCheck, Google, Census)
  • Direct feed into the DASSB model for calculation and scoring
  • Seamless transition to underwriting when a site is selected

Property Types Supported

P

Prospective Site

Available land or space being evaluated for potential acquisition or lease.

C

Competitor Location

Operating retail location used for competitive intelligence and market analysis.

E

Client Existing Location

Current operating locations in the client's portfolio for performance tracking.

B

Benchmark Comparable

Reference properties used for establishing performance benchmarks and projections.

Data Processing Pipeline

  1. Data Ingestion — Raw data is collected from source systems via APIs or manual entry
  2. Validation — Data quality rules ensure completeness and accuracy
  3. Normalization — Raw metrics are transformed into comparable units
  4. Enrichment — Calculated fields are derived from base data
  5. Scoring — DASSB model applies weighted criteria to generate composite scores
  6. Output — Rankings, projections, and recommendations are produced
Data Freshness Matters

Some data fields have time-sensitive validity. Traffic counts older than 3 years, for example, should be flagged for refresh in Module 13 (Data Quality).

Competitive Pressure Assessment

Optimal Positioning Zone

Analysis shows 1.0-1.5 miles from primary competitors is often the "sweet spot" — close enough to validate demand, far enough to minimize direct competition.

Distance Band Competitive Pressure Strategic Implication
< 0.5 miles Very High Direct competition, requires strong differentiation
0.5 - 1.0 miles High Shared trade area, validated demand
1.0 - 1.5 miles Optimal Demand validation without direct overlap
1.5 - 2.0 miles Moderate Partial overlap, separate primary trade areas
> 2.0 miles Low Independent trade areas, may be unvalidated
