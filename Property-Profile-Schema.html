The Property Profile as Data Foundation

The Master Property Profile serves as the standardized data container for every location in the Windfield analytical ecosystem.

Purpose

Whether a property is a prospective site, competitor location, client existing location, or benchmark comparable—it receives the same structured data treatment. This enables:

  • Apples-to-apples comparison across disparate property types
  • Consistent data ingestion from multiple sources (CoStar, CenterCheck, Google, Census)
  • Direct feed into the DASSB model for calculation and scoring
  • Seamless transition to underwriting when a site is selected

Property Types Supported

P

Prospective Site

Available land or space being evaluated for potential acquisition or lease.

C

Competitor Location

Operating retail location used for competitive intelligence and market analysis.

E

Client Existing Location

Current operating locations in the client's portfolio for performance tracking.

B

Benchmark Comparable

Reference properties used for establishing performance benchmarks and projections.

Data Processing Pipeline

  1. Data Ingestion — Raw data collected from source systems via APIs
  2. Validation — Data quality rules ensure completeness and accuracy
  3. Normalization — Raw metrics transformed into comparable units
  4. Enrichment — Calculated fields derived from base data
  5. Scoring — DASSB model applies weighted criteria
  6. Output — Rankings, projections, and recommendations produced
Data Freshness Matters

Traffic counts older than 3 years should be flagged for refresh in Module 13 (Data Quality).
